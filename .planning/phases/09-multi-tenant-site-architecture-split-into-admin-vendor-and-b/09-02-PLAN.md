# Vendor Portal Architecture

## 1. Create Vendor-specific Layout

Create `src/ProcureFlow.Vendor/Components/Layout/VendorLayout.razor`:
- Clean header with vendor logo placeholder
- Simplified navigation for vendor workflows
- Footer with company contact info
- Color scheme: Teal primary (#0D9488), warm gray accents (#78716C), light backgrounds
- Modern, approachable design suitable for external stakeholders
- Responsive design for mobile access

## 2. Create Vendor Pages Structure

Create directory `src/ProcureFlow.Vendor/Components/Pages/Vendor/`:
- `Dashboard.razor` - Vendor KPI overview, pending actions
- `RfpList.razor` - Available RFPs from invited buyers
- `BidCreate.razor` - Create/edit bid with line items
- `BidList.razor` - Submitted bids status
- `Contracts.razor` - Contracts awaiting signature
- `ContractSign.razor` - Digital signature flow

## 3. Create Vendor Shared Components

Create `src/ProcureFlow.Vendor/Components/Shared/`:
- `VendorNavMenu.razor` - Simplified navigation
- `RfpCard.razor` - RFP summary card for list view
- `BidForm.razor` - Bid entry form with validation
- `ContractCard.razor` - Contract status display
- `SignaturePad.razor` - Digital signature component

## 4. Create Vendor Program.cs

Create `src/ProcureFlow.Vendor/Program.cs`:
- Blazor Web App with SSR
- Register Vendor API endpoints
- Configure vendor-specific auth (API key or OAuth)
- Vendor-specific DI services

## 5. Create Vendor App.razor and Routes

Create routing with `/vendor/*` paths.
Include public landing page for vendor onboarding request.
