<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| authenticated principal to authorization middleware | Identity claims are interpreted for privileged access decisions. |
| API write handlers to audit subsystem | Mutation events cross into persistence and audit logging pipelines. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-13 | S | role claim parsing | mitigate | Validate authenticated principal and accepted role claims before policy resolution; reject malformed claim sets. |
| T-01-14 | T | audit metadata fields | mitigate | Use interceptor-controlled stamping and ignore client-provided audit fields to prevent tampering. |
| T-01-15 | R | mutation accountability | mitigate | Emit immutable structured audit events with actor, entity, operation, and UTC timestamp on every write. |
| T-01-16 | I | audit event payload | mitigate | Limit event content to identifiers and operation metadata, excluding sensitive payload bodies. |
| T-01-17 | D | authorization middleware | mitigate | Fail closed on policy lookup errors and short-circuit unauthenticated requests early to reduce load. |
| T-01-18 | E | endpoint privilege boundaries | mitigate | Enforce explicit per-endpoint policy declarations and add integration tests for role escalation attempts. |
</threat_model>
