## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client to admin API | Untrusted request payloads cross into privileged create/update handlers. |
| web API to database | Service account writes business-critical identity records. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | S | admin company/employee endpoints | mitigate | Require authenticated admin principal before route execution and reject anonymous or non-admin identities. |
| T-01-02 | T | company/employee update payloads | mitigate | Validate DTO shape, enum bounds, and immutable keys; reject overposting and malformed fields at endpoint boundary. |
| T-01-03 | R | create/update operations | mitigate | Persist actor id and UTC timestamps in audit columns for every write transaction. |
| T-01-04 | I | employee detail/list responses | mitigate | Scope employee reads by company boundary and return only required fields in DTO contracts. |
| T-01-05 | D | list endpoints | accept | Apply pagination defaults and max page size caps; residual risk accepted for v1 due bounded admin traffic. |
| T-01-06 | E | status update endpoint | mitigate | Enforce policy checks and server-side allowed transition logic before update commit. |
</threat_model>
